# Erste Release-Checkliste

## Repository

- Repository `DieListe01/IFeelDumpQuiz` anlegen
- Projektdateien hochladen
- `.gitignore` pruefen

## Vor dem ersten Release

- `VERSION` auf den gewuenschten Stand setzen
- Windows-Export lokal testen
- Installer lokal testen
- Update-Pruefung im Hauptmenue testen

## GitHub

- Release Notes vorbereiten
- Tag erstellen, z. B. `v0.1.0`
- GitHub Actions Workflow laufen lassen
- Pruefen, ob `IFeelDumpQuiz-win64.zip` am Release haengt

## Nach dem Release

- Installer auf einem sauberen Testsystem installieren
- App starten und Versionsanzeige pruefen
- Update-Ablauf gegen naechstes Testrelease pruefen
